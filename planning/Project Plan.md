### Project Plan


**Study Zen**

Sustainable Development Goal: SDG 3 – Good Health and Well-Being


**Project Overview**

The Stress & Study Balance Tracker is a mental health support system designed to help students manage academic stress while maintaining a healthy study routine. The system allows students to record study hours, track their stress levels, receive reminders to take breaks, and access relaxation tips or emergency support resources.

This project supports Sustainable Development Goal 3 (Good Health and Well-Being) by encouraging healthier study habits and helping students monitor their mental wellbeing.



**Project Objectives**

The main objectives of the system are:
1. Help students monitor study habits and stress levels
2. Encourage healthy study routines
3. Reduce academic burnout
4. Provide mental health support resources
5. Improve time management and productivity


**Key System Features**

The prototype includes the following core features:
_Study Hours Tracking_
1. Students can record daily study hours
2. Students can view study history and trends


_Stress Level Tracking_
1. Students can record their daily stress level
2. The system stores stress data for monitoring patterns


_Dashboard_
1. Displays study hours and stress level overview
2. Provides quick access to system features


_Reminder Notifications_
1. Study reminders
2. Break reminders to prevent burnout


_Relaxation Tips_
1. Breathing exercises
2. Short break suggestions
3. Stress reduction activities
4. Emergency Hotline Support
5. Access to mental health hotline contacts
6. Access to university counselling services


_AI Suggestions_
1. The system provides recommendations based on study hours and stress levels

_Admin Features_
1. Manage user accounts
2. Monitor system usage
3. Update mental health resources
4. Generate system reports




**Development Methodology**

The project follows an Agile development using the Scrum approach with two development sprints. Tasks are managed through a Trello Kanban board, allowing the team to track progress and organise development activities. Tasks move through these columns as the project progresses.

The Trello board includes the following columns:
1. Product Backlog
2. Sprint 1
3. Sprint 2
4. To Do
5. Doing
6. Done


**Development Timeline**

_Week 1 – Planning (11 Feb – 17 Feb)_
Activities:
1. Form project team and assign roles
2. Choose SDG topic and project idea
3. Define project goals and objectives
4. Identify stakeholders
5. Create user stories
6. Create product backlog
7. Set up Trello board
8. Create GitHub repository


_Week 2 – Sprint 1 (Design & Early Prototype) (18 Feb – 24 Feb)_
Activities:
1. Identify functional and non-functional requirements
2. Create Use Case Diagram
3. Create Class Diagram
4. Design system architecture
5. Create UI wireframes
6. Develop login prototype
7. Create system test plan


_Week 3 – Sprint 2 (Build & Testing) (25 Feb – 3 Mar)_
Activities:
1. Implement study hours tracking
2. Implement stress level tracking
3. Implement reminder notifications
4. Conduct usability testing
5. Collect feedback and bug reports
6. Update Trello progress



_Week 4 – Finalisation (4 Mar – 13 Mar)_
Activities:
1. Improve prototype based on feedback
2. Write report sections
3. Compile final report
4. Prepare presentation slides
5. Collect GitHub and Trello evidence
6. Final project submission
